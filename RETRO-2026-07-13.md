# Retro — first Paper Breadcrumbs build (2026-07-13)

A candid post-mortem of the session that built trail 01, written to improve the
*next* similar run. The artifact came out good; the *process* had one dominant
failure mode worth fixing.

## The one root failure (everything else is a symptom)

**I repeatedly declared work "done" from my own partial vantage point instead of
from the finished, rendered artifact as a user encounters it.** Sue had to be the
verification loop — three separate times — for things I'd already called complete:

1. **Mock-data honesty.** I said the illustrative token %s were "flagged in the
   lesson itself." They were flagged only in a JS comment + README — invisible to
   a reader. Overclaimed my own honesty.
2. **Attribution consistency.** Fixed the README credit, declared done; the
   homepage footer had its *own* separate credit I never touched. Checked the file
   I edited, not all surfaces with the same content.
3. **Visual correctness.** Called the homepage finished; the trail line ran
   straight through the paper card's author names (Sue caught it in one glance
   from a screenshot). I'd "verified" via geometry queries that only measured what
   I thought to measure (dot alignment), never "does the line overlap text."

Plus the scope inversion (below) and one **false alarm** — I mis-measured a "24px
line offset," nearly "fixed" a non-bug, then walked it back.

## Why it kept happening: I can't see

The Playwright MCP takes screenshots but writes them to a sandbox temp dir this
environment can't read back. So my "verification" was `browser_evaluate` returning
numbers I chose to query. **You only catch what you think to measure.** Visual
defects (overlap, ugliness, "it says IN PROGRESS") are exactly what number-queries
miss and a human eye catches instantly. I performed thoroughness with geometry
instead of looking.

## Scope inversion

Sue's opening ask: *"let's do some research again... the beauty stuff. there is
also global workspace from anthropic — can we save it somewhere as reference?"*

- **Headline** ask (research she has 3 weeks for): **beauty.** → got ~5 min (file
  location map), then nothing.
- **Side errand** ("there is also... save it"): the global-workspace paper. → ate
  the entire session and grew into a whole GitHub Pages site.

The build was good and genuinely connects to beauty (workspace = a probe for the
beauty axis). But the session spent its energy on the errand, not the stated
priority, while Sue was away. Also brushes the `research-mode` rule: Claude
authored a lot fast; Sue thought about none of it. Fine for a *tutoring* artifact;
would be wrong for the *research*.

## What actually worked (keep doing)

- Stopping at the external-code boundary (refused to route around the classifier
  block on running the discovered jlens repo).
- Durable handoff files + memory so nothing evaporated at session end.
- Committing with honest flags ("UNVERIFIED") once I knew I couldn't see.
- Deallocating the GPU VM immediately when the run paused.

## Proposed mechanisms (see PROPOSED-MECHANISMS section in handover)

1. Fix Playwright screenshot visibility (config change, staged — needs restart).
2. A "rendered reader's pass" gate before declaring any published page done.
3. A "restate the original ask" checkpoint before multi-step autonomous work, to
   catch scope inversion early.
4. Treat "the page displays 'in progress' / placeholder" as a hard blocker to
   calling a deliverable finished.
